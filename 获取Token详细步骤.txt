步骤1：访问Token管理页面
────────────────────────────────────────────────────
✓ 已自动打开：https://github.com/settings/tokens

如果没有打开，请手动访问上面的链接

步骤2：生成新Token
────────────────────────────────────────────────────
1. 点击右上角的 "Generate new token" 按钮
2. 选择 "Generate new token (classic)"
   ⚠️ 不要选择 Fine-grained tokens

步骤3：填写Token信息
────────────────────────────────────────────────────
在打开的页面中填写：

┌────────────────────────────────────────────────┐
│ Note (备注名称)                                │
│ ─────────────────────────────────────────────  │
│ Snake Game Deploy                              │
│                                                │
│ 或者：hanxhan000-snake-game-token              │
└────────────────────────────────────────────────┘

┌────────────────────────────────────────────────┐
│ Expiration (有效期)                            │
│ ─────────────────────────────────────────────  │
│ 推荐选择：90 days（90天）                     │
│                                                │
│ 其他选项：                                     │
│ - 30 days（30天）                             │
│ - 60 days（60天）                             │
│ - Custom（自定义）                            │
│ - No expiration（永不过期，不推荐）           │
└────────────────────────────────────────────────┘

步骤4：选择权限（重要！）
────────────────────────────────────────────────────
向下滚动，找到 "Select scopes" 部分

必须勾选：

  ✅ repo（完整的仓库控制权限）
     ├── ✓ repo:status
     ├── ✓ repo_deployment
     ├── ✓ public_repo
     └── ✓ repo:invite

⚠️ 只需要勾选 "repo" 这一项，它会自动包含所有子项

其他权限（可选，本项目不需要）：
  ☐ workflow
  ☐ write:packages
  ☐ delete:packages
  ☐ admin:org
  ... （其他都不需要勾选）

步骤5：生成Token
────────────────────────────────────────────────────
1. 滚动到页面底部
2. 点击绿色按钮 "Generate token"

步骤6：复制Token（⚠️ 非常重要！）
────────────────────────────────────────────────────
⚠️ Token 只会显示一次！

1. 页面会显示一个绿色的Token（类似这样）：
   ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

2. 点击右侧的 📋 复制按钮
   或者手动选中全部Token文字并复制

3. ⚠️ 立即保存到安全的地方：
   - 记事本
   - 密码管理器
   - 安全的文档

4. ⚠️ 不要关闭页面，直到确认已保存！

════════════════════════════════════════════════════
   ✅ Token格式检查
════════════════════════════════════════════════════

正确的Token格式：
  ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
  
特征：
  - 以 "ghp_" 开头
  - 后面跟随一串随机字符
  - 总长度约40个字符
  - 没有空格
  - 只包含字母、数字和下划线

如果你的Token不是这个格式，请重新生成！

════════════════════════════════════════════════════
   ⚠️ 常见问题
════════════════════════════════════════════════════

Q1: Token复制后去哪了？
A1: 已复制到剪贴板，可以直接Ctrl+V粘贴

Q2: 忘记保存Token怎么办？
A2: Token只显示一次，忘记了需要重新生成新Token
    （删除旧Token，生成新的）

Q3: Token过期了怎么办？
A3: 重新访问 https://github.com/settings/tokens
    生成新Token替换旧Token

Q4: 可以同时有多个Token吗？
A4: 可以！每个Token可以用于不同用途

Q5: Token会被别人看到吗？
A5: 不会！Token只在生成时显示一次
    之后GitHub也看不到完整Token
